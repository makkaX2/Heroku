# Heroku Userbot

Юзербот для Telegram, рассчитанный на деплой из GitHub в Dokploy через Nixpacks.

## Деплой в Dokploy

1. Создайте приложение Dokploy из этого репозитория.
2. Выберите Nixpacks как способ сборки.
3. Установите переменные окружения:
   - `HEROKU_DEPLOYMENT=dokploy`
   - `HEROKU_DATA_ROOT=/data/heroku` или другой постоянный путь
   - `API_ID`
   - `API_HASH`
4. Подключите постоянный volume по тому же пути, что и `HEROKU_DATA_ROOT`.
5. Запустите приложение. Стартовая команда: `python -m heroku`.

Приложение больше не клонирует само себя и не ставит зависимости во время запуска. Все зависимости должны собираться на этапе build.

Если Dokploy продолжает падать на этапе export образа, отключите build cache в Dokploy и запустите redeploy с нуля. Ошибка вида `crc32 mismatch` обычно указывает на проблему Docker/build cache на сервере, а не на ошибку в коде приложения.

## Локальная разработка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m heroku
```

Если нужно хранить конфиг, сессии и загруженные модули вне репозитория, задайте `HEROKU_DATA_ROOT` перед запуском.

## Структура проекта

- `heroku/` основной код, команды, веб-интерфейс и утилиты
- `assets/` текстовые и шрифтовые ресурсы
- `web-resources/` шаблоны, статические файлы и шрифты
- `nixpacks.toml` конфигурация сборки и старта для Nixpacks

## Безопасность

Сторонние модули и команды выполнения кода рассматривайте как потенциально опасные. Проверяйте источники и не доверяйте неизвестным `.terminal`, `.eval` и загрузчикам модулей.
